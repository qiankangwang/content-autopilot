# Content Examples — Patterns by Platform

Illustrative patterns for an "AI 工具实操" style account. These show the *shape* of
a good post on each platform, not fixed templates — always read the room first.

## Xiaohongshu — 效率工具 note

Key patterns:
- **Title**: pain point + solution + one emoji.
- **Cover text**: 3 punchy benefit lines.
- **Body**: 1️⃣2️⃣3️⃣ structure, each step ending with a concrete "怎么用".
- **Voice**: conversational, lightly self-deprecating.
- **CTA**: invite a comment ("评论区聊聊你的"), not a spammy "扣1私你".

## Xiaohongshu — 实测对比 note

Key patterns:
- A concrete number in the title (time saved, price, count).
- A difficulty rating (★☆☆) so the reader knows the effort.
- An honest disclaimer ("不是万能") — builds trust.
- Modest, specific claims over hype.

## Zhihu — architecture analysis answer

Key patterns:
- **Bold conclusion first**, then justify it.
- **Analogy frame** to make an abstract comparison concrete.
- **Benchmark data upfront** for credibility (name the benchmark + the score).
- **A few key dimensions**, each with a concrete example.
- **Dual comparison**: hold one variable fixed, vary the other, report the gap.
- **Actionable advice**: 3 concrete tips the reader can use tomorrow.
- **Readable structure**: tables for data, numbered sections, bold takeaways.
- **Target length**: ~2000–2500 Chinese characters (within the 1000–3000 range).

## Douyin — short demo video

Key patterns:
- **Hook in 3 seconds**: rhetorical question + surprising answer.
- **Show, don't tell**: a screen-recording-style demo beats narration alone.
- **Time-anchored**: `[0-3秒]` `[3-8秒]` beats keep it tight.
- **One light humor twist** to make it memorable.
- **Engagement via a question**, not "扣1私你工具名".
